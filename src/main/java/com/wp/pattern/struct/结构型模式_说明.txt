    装饰器模式
    {
        英文/其他叫法: Decorator Pattern
        思想: 允许向一个现有的对象添加新的功能，同时又不改变其结构
        实现: 这种模式创建了一个装饰类，用来包装原有的类，
            并在保持类方法签名完整性的前提下，提供了额外的功能。
        参考资料：https://www.runoob.com/design-pattern/decorator-pattern.html
    }
